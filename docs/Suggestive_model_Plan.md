# FLASH+ 実装仕様書

**遠距離精度向上のための追加モジュール設計**
ベースモデル：FLASH（arXiv:2511.07377）

---

## 0. 概要

本文書は、FLASHのLiDAR超解像モデルに対して遠距離（30〜60m）精度を向上させるための2つの追加モジュールと、それに対応する損失関数、およびアブレーション実験の設計を定義する。ベースとなるFLASHの完全再現実装は既に完了しているものとし、本書に記載の差分のみを実装すること。

**問題の背景：** FLASHはKITTIデータセットにおいて近距離（0〜30m）でMAE=0.239mと高精度を達成する一方、遠距離（30〜60m）ではMAE=2.045m（約8.6倍）・IoU=0.105（近距離の約4分の1）と大幅に精度が低下する。この劣化の根本原因は、現在のFrequency-Aware Attention（FA）モジュールが全距離で同一の固定周波数フィルタを使用している点にある。遠距離では点群密度が低下し、高周波成分がエイリアシングノイズとして作用するため、近距離向けに最適化されたフィルタは適切でない。

追加するモジュールは以下の2つ。

- **Range-Adaptive Frequency Kernel（RAFK）**：距離に応じた周波数フィルタの動的生成
- **Meta-Kernel Discriminator（MKDisc）**：学習時のみ使用する幾何制約付き敵対的損失

推論時のアーキテクチャ変更はRAFKのみであり、MKDiscは学習時のみ使用する。推論速度（66 FPS）は維持すること。

---

## 1. Range-Adaptive Frequency Kernel（RAFK）

### 1.1 動機

LiDARの点群密度はセンサーからの距離の2乗に反比例するため、サンプリングのNyquist周波数が距離依存で変化する。近距離では高周波成分が実際のエッジ・物体境界の信号であるが、遠距離ではスパースサンプリングによるエイリアシングノイズである。FLASHの既存FAは式(7)において単一の固定Convを全距離で共有しており、この矛盾した処理が遠距離精度劣化の一因となっている。

| 距離帯 | 高周波成分の意味 | 適切な処理 |
|--------|----------------|-----------|
| 近距離（密） | 実際のエッジ・物体境界の信号 | 高周波を通過させる |
| 遠距離（疎） | エイリアシング・補間アーティファクト | 高周波を減衰させる |

### 1.2 変更箇所

**対象：** FAモジュールの周波数ブランチ（式7のConv部分のみ）
**変更範囲：** 式(6)〜(9)のうち、式(7)のみを以下に置き換える。式(6)(8)(9)は変更不要。

#### 変更前

```
X_freq     = FFT2D(mean(X, dim=C))            # 式(6) ← 変更なし
F_attn     = σ(Conv(|X_freq|))                # 式(7) ← ここのみ変更
X_freq_out = iFFT2D(X_freq ⊙ F_attn)         # 式(8) ← 変更なし
Output     = Attn_spatial + α·expand(X_freq_out)  # 式(9) ← 変更なし
```

#### 変更後（式7の置き換え）

```python
# ── ステップ1: ウィンドウwの距離推定特徴量を計算 ──────────────────────────
#
# m_i = 1 if pixel i is valid (r > 0) else 0
n_valid  = Σ m_i                                      # 有効ピクセル数
r_bar_w  = Σ(r_i · m_i) / max(n_valid, 1)            # 加重平均距離
#   └ n_valid == 0 の場合は r_bar_w = r_max (= 60m) をデフォルト値として使用

# ── ステップ2: 3特徴量の入力ベクトルを構成 ───────────────────────────────
feat_w = [
    v_w / H,              # 行インデックス（安定した代理変数）
    r_bar_w / r_max,      # 実測平均距離（信頼できる場合の精密推定）
    n_valid / n_total     # 有効点率（上記2特徴の信頼度重み）
]
# v_w     : ウィンドウwの中心行インデックス（0-indexed）
# H       : range imageの行数（= 64 for KITTI）
# r_max   : 最大距離（= 60.0 m）
# n_total : ウィンドウ内の全ピクセル数（= 2×8 = 16）

# ── ステップ3: 混合係数の生成 ─────────────────────────────────────────────
α_blend = MLP_α(feat_w)
# MLP_α: Linear(3→16) → ReLU → Linear(16→1) → Sigmoid
# α_blend = 1: Conv_near 優勢（高周波通過）
# α_blend = 0: Conv_far  優勢（低周波通過）

# ── ステップ4: ソフト混合による周波数フィルタ（式7の置き換え） ─────────────
F_attn = σ( α_blend · Conv_near(|X_freq^w|)
           + (1 - α_blend) · Conv_far(|X_freq^w|) )
```

### 1.3 3特徴量設計の意図

`n_valid / n_total` を第3特徴量として含めることで、MLPが以下の挙動を自然に学習する。

- `n_valid` が大きい（密な近距離）→ `r_bar_w` を信頼 → 実測距離で周波数適応
- `n_valid` が小さい（疎な遠距離）→ `r_bar_w` を不信頼 → 行インデックス `v_w/H` にフォールバック

`r_bar_w = r_max` というデフォルト値も `n_valid=0` という入力と併せて渡されるため、MLPが「距離推定が信頼できない状況」であることを明示的に認識できる。

### 1.4 追加パラメータ仕様

| パラメータ | 設定値 | 備考 |
|-----------|--------|------|
| MLP_α 構造 | `Linear(3→16)→ReLU→Linear(16→1)→Sigmoid` | 追加パラメータ数は65（無視できる規模） |
| Conv_near | FLASHオリジナルのConvと同一構造 | 近距離向け高周波通過フィルタとして学習 |
| Conv_far | FLASHオリジナルのConvと同一構造 | 遠距離向け低周波通過フィルタとして学習 |
| α_blend 初期値 | 0.5（MLP出力バイアス=0で初期化） | 学習初期は近遠均等混合から開始 |
| r_max | 60.0 m | FLASHのlog(r+1)変換後の最大値に合わせて正規化 |

> **注意：** FLASHはrange値を `log(r+1)` 変換して保存している。`r_bar_w` の計算にはピクセル値を `exp(v)-1` で逆変換した実距離（m）を使用すること。

## 2. Meta-Kernel Discriminator（MKDisc）

### 2.1 動機

FLASHのL1損失のみの学習は、スパースな遠距離領域において「平均的な予測」への収束を促す。RangeLDM（arXiv:2403.10094）が提案したMeta-Kernelを利用したDiscriminatorは、3D球面座標から畳み込み重みを動的生成することで、生成されたrange imageが3D幾何構造として整合性を持つかどうかを評価できる。これを敵対的損失として追加することで、L1損失だけでは強制できない幾何的一貫性を学習に導入する。

> **重要：** MKDiscは学習時にのみ使用する。推論時にはロードせず、推論速度（66 FPS）を完全に維持すること。

### 2.2 Meta-Kernelの定式化

RangeLDMの式(5)(6)をそのまま採用する。

```
# 各点の特徴更新（Meta-Kernel）
h'_i = W( A_{j∈N(i)}( Φ(γ(p_j, p_i)) ⊙ h_j ) )

# 球面座標間の相対距離ベクトル（3次元）
γ(p_j, p_i) = { r_j·cos(Δθ)·cos(Δφ) - r_i,
                 r_j·cos(Δθ)·sin(Δφ),
                 r_j·sin(Δθ) }
Δθ = θ_j - θ_i,  Δφ = φ_j - φ_i
```

- `Φ`：2層MLP（全結合）
- `A`：連結操作
- `W`：全結合層
- `N(i)`：ピクセルiの2Dグリッド近傍（3×3推奨）

Discriminatorのアーキテクチャはpatch-based adversarial discriminator（PatchGAN形式）とし、標準的な畳み込み層をMeta-Kernel層に置き換えたものを使用する。実装の詳細はRangeLDMの公開コード（https://github.com/WoodwindHu/RangeLDM）を参照すること。

## 3. 損失関数

### 3.1 全体の損失

```
L_total = L_L1
        + λ₁ · w(r) · L_adv(MKDisc)
        + λ₂ · L_freq_consistency
```

### 3.2 各損失の定義

#### 3.2.1 L_L1（既存・変更なし）

FLASHオリジナルの損失（式3）をそのまま使用する。

```
L_L1 = ‖I_h - Î_h‖₁
```

#### 3.2.2 L_adv（敵対的損失・学習時のみ）

MKDiscを用いたGAN Hinge損失（RangeLDMの式7に準拠）。

```
L_adv = L_GAN_Hinge(Generator=FLASH, Discriminator=MKDisc)
```

**距離重み `w(r)`：** 遠距離での幾何制約をより強く課すための重み関数。

```
w(r) = 1 + β · max(0, r - r_near) / (r_far - r_near)
# r_near = 30m, r_far = 60m, β = 2.0（初期推奨値）
```

この設定では 0〜30m で重み=1.0、30〜60m で重みが 1.0→3.0 に線形増加する。βは検証セットのMAE（30-60m）を見ながら調整すること。

#### 3.2.3 L_freq_consistency（周波数一貫性正則化）

`Conv_near` と `Conv_far` が単に同一のフィルタに収束しないよう促し、実際に異なる周波数特性を学習することを保証する。

```
L_freq_consistency = -‖W_near - W_far‖_F
# W_near, W_far: それぞれ Conv_near, Conv_far の重み行列
# ‖·‖_F: フロベニウスノルム
# 負符号により、2つのカーネルが互いに異なる方向への学習を促進
```

### 3.3 ハイパーパラメータの初期推奨値

| パラメータ | 初期値 | 調整の目安 |
|-----------|--------|-----------|
| `λ₁` | 0.1 | MKDiscの効果が強すぎる場合は0.05に下げる |
| `λ₂` | 0.01 | Conv_nearとConv_farが収束する場合は0.05に増やす |
| `β`（距離重みの傾き） | 2.0 | 遠距離MAEの改善が停滞する場合は3.0に増やす |
| `r_near` | 30.0 m | FLASHのTable III境界に一致。変更不要 |
| `r_far` | 60.0 m | KITTIの最大計測距離に一致。変更不要 |

## 4. アブレーション実験設計

### 4.1 実験設定

以下の4設定を全てゼロから再学習する（FLASHの既存重みからのファインチューニングではなく完全再学習）。学習設定（データセット・エポック数・オプティマイザ・学習率スケジュール）はFLASHオリジナルと完全に同一とし、変数を追加モジュールのみに限定すること。

- **データセット：** KITTIデータセット（FLASHと同一分割）
  - 訓練：20,000フレーム（sequence 0以外）
  - テスト：2,000フレーム（sequence 0）
  - 入力：16チャンネル、出力：64チャンネル（4倍アップサンプリング）

### 4.2 アブレーション設定テーブル

| 設定名 | RAFK | MKDisc | 期待される役割 |
|--------|:----:|:------:|--------------|
| Baseline (FLASH) | ✗ | ✗ | 再現値（FLASH Table III と一致を確認） |
| + RAFK | ✓ | ✗ | RAFKの遠距離スパース適応効果を分離 |
| + MKDisc | ✗ | ✓ | 幾何的一貫性ロスの効果を分離 |
| **提案モデル（両方）** | ✓ | ✓ | 相乗効果の確認 |

### 4.3 評価指標

以下の指標を全設定で測定する（FLASHのTable IIIと対応させること）。

- MAE（m）：近距離（0〜30m）・遠距離（30〜60m）を分離して報告
- CD（Chamfer Distance）：近距離・遠距離を分離して報告
- IoU：近距離・遠距離を分離して報告
- F1スコア：近距離・遠距離を分離して報告
- 推論速度（FPS）：提案モデルのみ、FLASHとの比較で報告

> 距離分離の計算方法はFLASH論文 Section IV.A（Evaluation Metrics）の記述に従い、同一の実装を使用すること。

### 4.4 期待される結果と主張

アブレーション結果が示すべき主要な主張は以下の通り。

1. RAFKが遠距離MAEの主要な改善要因であること（`+ RAFK` 行の遠距離MAEがBaselineを有意に上回る）
2. MKDiscがIoU・CDの改善に寄与すること（幾何的一貫性の向上）
3. 両モジュールの組み合わせが相乗効果を持つこと（提案モデルが各単体を上回る）
4. 推論速度がFLASH（66 FPS）と同等を維持すること

## 5. 実装上の注意事項

### 5.1 学習の安定性

- MKDiscの追加により学習が不安定になる場合は、最初の10エポックは `L_L1` のみで学習し、その後 `λ₁` を0からウォームアップする形で段階的に導入することを検討する
- `L_freq_consistency` がNaNになる場合は `Conv_near`・`Conv_far` の重みにグラジェントクリッピング（max_norm=1.0）を適用すること

### 5.2 その他の実験・可視化

本書に記載のない実験（全体的な定量評価テーブル・定性的可視化・既存手法との比較等）はFLASH論文の Section IV（Experimental Results）をそのまま踏襲すること。本書はFLASHに対する差分のみを定義している。
